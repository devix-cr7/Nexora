# Ledger & Ops — المرحلة 1

نظام إدارة أعمال (مشاريع، عملاء، فواتير، مصاريف، مواعيد) — هذي المرحلة الأولى: الأساس + تسجيل الدخول.

## شغّل المشروع محلياً

```bash
npm install
npm run dev
```

يفتح على `http://localhost:5173`

## متغيرات البيئة

المفاتيح موجودة بملف `.env` (متصل بمشروعك على Supabase). **لا ترفع هذا الملف لـ GitHub** — هو موجود بـ `.gitignore` أصلاً.

لو رفعت المشروع لمكان جديد أو غيّرت مشروع Supabase، حدّث القيم بـ `.env`:

```
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```

## شنو موجود بهذي المرحلة

- تسجيل حساب جديد (بريد + كلمة مرور) مع تأكيد بالإيميل
- تسجيل دخول / خروج
- Layout رئيسي: Sidebar + صفحة رئيسية (Dashboard) فيها إحصائيات مؤقتة
- صفحات "قيد البناء" لكل من: المشاريع، العملاء، الفواتير، المصاريف، المواعيد (تنبني بالمراحل الجاية)
- ثيم غامق + دعم كامل للعربي RTL + خط IBM Plex Sans Arabic

## النشر على GitHub Pages

نفس أسلوبك المعتاد بـ GitHub Desktop:
1. سوي بناء الإنتاج: `npm run build`
2. محتوى مجلد `dist` هو اللي ينرفع
3. تأكد أن Source بإعدادات GitHub Pages مضبوط على الفرع/المجلد الصحيح

⚠️ **مهم:** ملف `.env` ما يترفع لـ GitHub (لأمانه)، فلو نشرت المشروع من جهاز/مكان جديد، لازم تضيف المتغيرات بطريقة مناسبة (مثلاً GitHub Actions secrets، أو نتكلم عن هذا بمرحلة النشر النهائي).
